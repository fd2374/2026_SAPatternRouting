# Crosstalk-Aware Package-Level Pattern Router

A Python-based package-level routing tool for multi-die packages with crosstalk-aware optimization under 135-degree (obtuse-angle only) routing constraints.

## Overview

This router generates candidate routing patterns for each net and selects the optimal combination using Simulated Annealing (SA), minimizing a cost function that considers:

- **Wirelength** — total wire length across all nets
- **Crosstalk** — coupling between parallel same-layer segments (coupling length / spacing model)
- **Via count** — number of layer transitions
- **Same-layer crossings** — penalized as large violations
- **DRC violations** — wire-via and via-via overlap checks

### Routing Constraints

All routes use **135-degree turns only** (horizontal, vertical, and 45-degree diagonal segments). Supported pattern types:

| Pattern | Description |
|---------|-------------|
| H-D-H / V-D-V | Homogeneous: Rectilinear → Diagonal → Rectilinear |
| D-H-D / D-V-D | Inverted: Diagonal → Rectilinear → Diagonal |
| V-D-H / H-D-V | Mixed: different rectilinear directions at each end |

Multi-layer variants are generated by inserting vias at bend points or arbitrary positions along segments.

## Project Structure

```
router/
├── __init__.py        # Package init
├── data_types.py      # Core data structures (Point, Segment, Net, Package, etc.)
├── parser.py          # Benchmark file parser (.gp, .iopad, .net, .bump)
├── pattern_gen.py     # 135-degree pattern generation
├── solver.py          # Simulated Annealing solver with incremental delta
├── util.py            # Crosstalk, crossing, DRC calculations (Numba-accelerated)
├── output.py          # Result file writer
└── visualizer.py      # Matplotlib visualization (routing plots, SA convergence)

benchmarks/            # Multi-die package routing benchmarks
test.py                # Multi-seed parallel benchmark runner
requirements.txt       # Python dependencies
```

## Quick Start

```bash
pip install -r requirements.txt

# Run on a benchmark
python -m router.main benchmarks/dense1

# With custom parameters
python -m router.main benchmarks/dense2 --beta 20 --max-iter 2000

# Skip visualization
python -m router.main benchmarks/dense1 --no-viz

# Compare with crosstalk-unaware baseline
python -m router.main benchmarks/dense1 --baseline

# Multi-seed parallel run (picks best result)
python test.py benchmarks/dense1 --seeds 8 --workers 4
```

### Key Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--alpha` | 1.0 | Wirelength weight |
| `--beta` | 10.0 | Crosstalk weight |
| `--gamma` | 0.0 | Via count weight |
| `--max-iter` | 5000 | SA iterations |
| `--seed` | 42 | Random seed |
| `--baseline` | off | Run crosstalk-unaware comparison |
| `--no-viz` | off | Skip visualization |

## Benchmark Format

Each benchmark directory contains four files (see `benchmarks/format.txt` for details):

- `*.gp` — Global parameters (package size, layers, wire/via width, spacing)
- `*.iopad` — IO pad definitions per die
- `*.net` — Net connectivity (2-pin chip-to-chip nets)
- `*.bump` — Bump locations

## Output

Results are written to `results/<benchmark_name>/`:

- `routes.txt` — Per-net routing details (segments, vias)
- `summary.txt` — Solution summary statistics
- `crosstalk.txt` — Pairwise crosstalk values
- `routing_overview.png` — Overview visualization with violation markers
- `routing_per_layer.png` — Per-layer routing view
- `crosstalk_heatmap.png` — Pairwise crosstalk heatmap

## Dependencies

- Python 3.8+
